## Context

Media Studio 当前菜单结构和功能划分无法满足业务增长需求。现有架构缺少任务管理、图文成果管理、灵活的平台配置等能力。本次重构涉及前端菜单、路由、视图模块、目录结构、配置文件等多个层面。

当前代码库位于纯前端扩展（无构建步骤，无框架，无 npm 依赖），所有数据以文件形式存储在 Hermes WebUI 工作空间中。

## Goals / Non-Goals

**Goals:**
- 重构侧边栏菜单为新的四级分组（生产流程、发布管理、资源管理、运营配置）
- 引入任务系统：支持素材任务和文案任务，手工和 Agent 两种模式
- 引入配置驱动状态机：任务生命周期在配置文件中定义，前端自动渲染
- 引入图文库：Markdown + .meta.json sidecar 存储，分片索引
- 引入平台配置：平台 CRUD + 自定义发布类型
- 引入发布记录：发布表单 + 历史清单
- 更新项目设计文档和 AGENTS.md 反映新架构

**Non-Goals:**
- 不涉及 API 推送发布（纯记录）
- 不做数据迁移（当前无有效数据）
- 不做数据统计看板替代方案
- 不涉及多用户权限系统

## Decisions

### D1: 任务存储 → 目录结构 `tasks/<uuid>/`

任务以独立子目录存储，每个任务一个 UUID 目录，包含：
- `.meta.json`：任务元数据（类型、模式、状态、流转历史等）
- `brief.md`：创作简报/提示词
- Agent 任务可能写入的中间数据

**理由**：与现有素材库的 sidecar 模式一致，方便 Agent 扫描目录读取/修改数据，无需额外的数据库依赖。

### D2: 配置驱动状态机 → `configs/workflows/task-lifecycle.json`

任务生命周期在 JSON 配置文件中定义，包含：
- 每个任务类型的状态列表、看板可见状态、合法流转、终止状态、卡片颜色
- 前端根据配置渲染看板列、状态切换按钮、卡片样式

**理由**：新增任务类型（如音频任务）只需加配置文件，零代码改动。比硬编码 if/else 更灵活。

### D3: 图文库 → 方案B（Markdown + .meta.json sidecar）

- 目录：`copywriting/YYYY/MM/<uuid>/content.md` + `.meta.json`
- Markdown 中引用素材库路径（如 `![](../assets/2026/07/11/xxx.jpg)`)
- 分片索引：`.index/copywriting/YYYY/MM/` 按月分片

**理由**：与现有素材库设计一致，索引复用 search.js 的 shard 模式。Markdown 保持可读性和可编辑性。

### D4: 平台配置 → 纯记录，不可删除

- 目录：`configs/platforms/<id>.json`
- 字段：名称、发布类型列表、可选 API 端点（预留）
- 平台不可删除（保护已有发布记录的引用完整性）
- 可编辑和禁用

**理由**：平台可能被发布记录引用，物理删除会导致数据不一致。

### D5: 菜单结构 → 重建 MENU_GROUPS

- 完全重写 `app.js` 中的 `MENU_GROUPS` 和 `ICONS`
- 新增视图注册：`tasks`、`publish`、`copywriting`、`platforms`
- 移除视图注册：`generation`、`themes`、`dashboard`、`package-editor`
- `calendar` 从发布管理移到资源管理分组下

### D6: 看板 → 任务卡片模式

- 看板只展示任务卡片（对应 `tasks/<uuid>/` 目录）
- 初始化状态的任务不展示在看板
- 列由 `task-lifecycle.json` 中的 `kanban_states` 动态生成
- 不同任务类型卡片使用不同颜色（素材=蓝色系，文案=绿色系）
- 删除顶部主题/时间/搜索筛选

### D7: 审批 → 区分任务类型

- 审批界面根据 `task.type` 展示不同内容
- 素材任务：图片/视频预览 + 审批操作
- 文案任务：Markdown 渲染预览 + 审批操作

## Risks / Trade-offs

- **[风险] 配置驱动状态机增加复杂度**：需要前端解析 JSON 配置文件，校验状态流转合法性。→ **缓解**：状态机逻辑简单（单向流转为主），解析代码可控制在 100 行以内。
- **[风险] 任务目录增长**：大量任务可能导致 `tasks/` 目录下子目录过多。→ **缓解**：使用 UUID 避免命名冲突，可考虑按月分片目录（如 `tasks/YYYY/MM/<uuid>/`）。
- **[风险] 图文库 Markdown 图片引用路径**：移动素材库文件可能导致文案中图片失效。→ **缓解**：素材文件一旦完成不移动（现有设计已如此），路径引用稳定。
- **[权衡] 任务与素材不直接关联**：用户无法从素材库追溯到产生它的任务。→ 已在设计中和用户确认，当前不需要此功能。
